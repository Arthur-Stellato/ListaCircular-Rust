02 — Linguagem

Sintaxe Básica
do Rust

Variáveis & Mutabilidade

Em Rust toda variável é imutável por padrão. Para permitir mudanças, declare com mut.

let x = 5; // imutável – não pode mudar let mut y = 10; // mutável – pode mudar y = 20; // ✓ funciona x = 15; // ✗ erro! x é imutável

Funções

Declaradas com fn. O tipo de retorno vem após ->. A última expressão sem ponto-e-vírgula é retornada implicitamente.

fn maior(a: i32, b: i32) -> i32 { if a > b { return a; // retorno antecipado } b // retorno implícito }

null vs None — A grande diferença do Rust

Em linguagens como C, Java e JavaScript, null é um valor especial atribuível a qualquer ponteiro ou referência — e acessar algo nulo em tempo de execução causa crashes (NullPointerException, segfault). Rust elimina null completamente: em seu lugar existe None, uma variante do tipo Option<T> que o compilador obriga você a tratar.

outras linguagens — null perigoso
// Java / C / JavaScript String nome = null; // crash silencioso em tempo de execução: nome.length(); // NullPointerException ! // C int *p = NULL; *p = 42; // segmentation fault ! // o compilador NÃO avisa — explode em produção
Rust — None seguro e explícito
// Rust — o tipo já diz que pode ser vazio let nome: Option<&str> = None; // o compilador FORÇA o tratamento: match nome { Some(n) => println!("olá, {}", n), None => println!("sem nome"), } // esquecer o None = erro em compilação ✓

Resumo: None não é um ponteiro nulo — é um valor de tipo seguro. O compilador garante que você nunca acessa o conteúdo de um Option sem antes verificar se ele é Some. Isso elimina toda uma categoria de bugs que causam falhas em sistemas críticos escritos em C ou Java.

Conceitos-chave para a Lista Circular

struct

Agrupa dados relacionados. Define o nó e a lista em si.

impl

Adiciona métodos à struct: inserir, percorrer, editar, remover.

Option<T>

Tipo que representa "algo que pode existir ou não". Possui duas variantes: Some(v) quando há valor, e None quando ausente.

Some(v)

Variante de Option que contém um valor. Usado ao inserir nós: Some(valor) sinaliza que aquele slot está ocupado.

match

Desmonta um Option com segurança, forçando o tratamento explícito de Some e None.

04 — Estrutura de Dados

Lista Circular
Estática

O que é?

Diferente de uma lista encadeada simples, na lista circular o último nó aponta de volta para o primeiro. Não existe um "fim" real — qualquer nó pode ser o ponto de partida.

Nossa implementação usa dois arrays de tamanho fixo como backing store: nos[] guarda os valores e prox[] guarda os índices dos sucessores — sem alocação dinâmica.

Estrutura interna

const MAX: usize = 5; // capacidade máxima struct ListaCircular<T: Copy + Display + PartialEq> { nos: [Option<T>; MAX], // armazena os valores prox: [usize; MAX], // índice do próximo nó cabeca: Option<usize>, // índice do primeiro nó tamanho: usize, // elementos ativos }

Inserção — Início vs. Fim

Ambas as operações encontram um slot livre no array, inserem o valor e ajustam os ponteiros — a diferença está em onde a cabeça é atualizada.

inserir_inicio

Inserir no Início

O novo nó vira a nova cabeça. Seu prox aponta para a cabeça anterior, e a cauda passa a apontar para ele.

pub fn inserir_inicio(&mut self, valor: T) { if self.tamanho == MAX { println!(" lista cheia (máximo: {})", MAX); return; } let slot = self.achar_slot_livre().unwrap(); self.nos[slot] = Some(valor); if self.tamanho == 0 { self.prox[slot] = slot; self.cabeca = Some(slot); } else { let cabeca = self.cabeca.unwrap(); let cauda = self.achar_cauda(); self.prox[slot] = cabeca; self.prox[cauda] = slot; self.cabeca = Some(slot); } self.tamanho += 1; println!("{} inserido no início", valor); }
Antes:  HEAD→[20]→[30]→(HEAD)
Depois: HEAD→[10]→[20]→[30]→(HEAD)
VS
inserir_fim

Inserir no Fim

A cabeça não muda. O novo nó é encaixado após a cauda atual: a cauda aponta para éle, e ele fecha o ciclo apontando para a cabeça.

pub fn inserir_fim(&mut self, valor: T) { if self.tamanho == MAX { println!("lista cheia (máximo: {})", MAX); return; } let slot = self.achar_slot_livre().unwrap(); self.nos[slot] = Some(valor); if self.tamanho == 0 { self.prox[slot] = slot; self.cabeca = Some(slot); } else { let cabeca = self.cabeca.unwrap(); let cauda = self.achar_cauda(); self.prox[slot] = cabeca; self.prox[cauda] = slot; } self.tamanho += 1; println!("{} inserido no fim", valor); }
Antes:  HEAD→[10]→[20]→(HEAD)
Depois: HEAD→[10]→[20]→[30]→(HEAD)

Demais Métodos

ler_proximo(pos)
Navega até a posição dada e retorna o valor do nó seguinte via Option<T>. Se a posição for a cauda, o "próximo" é a própria cabeça — demonstrando a circularidade.
pub fn ler_proximo(&self, pos: usize) -> Option<T> { if self.tamanho == 0 || pos >= self.tamanho { return None; } let idx = self.idx_na(pos); if self.prox[idx] == self.cabeca.unwrap() { println!("próximo de [{}] é a cabeça da lista", pos); } self.nos[self.prox[idx]] }
percorrer()
Imprime todos os nós partindo da cabeça. Um loop for i in 0..tamanho garante exatamente uma volta — evita loop infinito sem quebrar a estrutura circular. Também exibe os índices físicos de cada slot.
pub fn percorrer(&self) { if self.tamanho == 0 { println!("lista vazia"); return; } // primeira passagem: valores print!(" HEAD → "); let mut idx = self.cabeca.unwrap(); for i in 0..self.tamanho { print!("[{}]", self.nos[idx].unwrap()); if i < self.tamanho - 1 { print!(" → "); } idx = self.prox[idx]; } println!(" → (HEAD)"); // segunda passagem: índices físicos print!(" "); idx = self.cabeca.unwrap(); for i in 0..self.tamanho { print!(" {} ", idx); if i < self.tamanho - 1 { print!(" "); } idx = self.prox[idx]; } println!(); println!(" tamanho: {}/{}", self.tamanho, MAX); }
remover(pos)
Trata três cenários: posição inválida (retorna None), único elemento (zera a lista), ou caso geral (faz o antecessor pular o nó removido). Se remover a cabeça, atualiza o ponteiro cabeca.
pub fn remover(&mut self, pos: usize) -> Option<T> { // CENÁRIO A — posição inválida if pos >= self.tamanho { return None; } // CENÁRIO B — lista com UM único elemento if self.tamanho == 1 { let idx = self.cabeca.unwrap(); let valor = self.nos[idx].take(); self.cabeca = None; self.tamanho = 0; return valor; } // CENÁRIO C — lista com 2+ elementos let (rem, ant) = if pos == 0 { (self.cabeca.unwrap(), self.achar_cauda()) } else { let a = self.idx_na(pos - 1); // acha o antecessor (self.prox[a], a) }; self.prox[ant] = self.prox[rem]; // pula o nó removido if pos == 0 { self.cabeca = Some(self.prox[rem]); // atualiza cabeça } let valor = self.nos[rem].take(); self.tamanho -= 1; valor }
tabela_fisica()
Exibe o estado raw dos dois arrays internos: índice físico, valor armazenado (ou vazio), ponteiro prox, e marcação de HEAD. Útil para depurar a estrutura interna e entender como os slots se relacionam.
pub fn tabela_fisica(&self) { println!(" idx │ nos │ prox"); println!(" ────┼──────────┼──────"); for i in 0..MAX { let val = match self.nos[i] { Some(v) => format!("{:>8}", v), None => " vazio ".to_string(), }; let head = if self.cabeca == Some(i) { " ← HEAD" } else { "" }; println!(" {} │ {} │ {} │{}", i, val, self.prox[i], head); } }

05 — Aplicações

Conclusão &
Onde é Usado

Lista Circular — Aplicações Reais

A estrutura circular permite navegar indefinidamente sem precisar reiniciar o índice, o que a torna ideal em cenários onde a rotação contínua é essencial.

⏱ Sistemas Operacionais

Escalonamento de processos por Round-Robin: a CPU visita cada processo na fila circular, garantindo que nenhum seja esquecido indefinidamente.

🎮 Jogos & Animações

Gerenciamento de sprites ciclados, inventários em loop, sequências de animação frame-a-frame e trilhas de áudio em repetição contínua.

📡 Redes & Protocolos

Buffers de anel (ring buffers) em comunicação serial, redes TCP/IP e streaming de áudio/vídeo, evitando realocações de memória.

🖥 Interfaces & UI

Carrosséis de imagens, playlists de música, histórico de desfazer/refazer (undo/redo) circular com tamanho fixo.

Rust — Onde a Linguagem Brilha

Rust combina desempenho de C/C++ com segurança de memória garantida pelo compilador — sem coletor de lixo. Isso o torna atraente para domínios onde falhas custam muito.

Sistemas & Embarcados
  • Kernels & OS — partes do kernel Linux já aceitam Rust; o sistema Redox OS é 100% Rust
  • Firmware IoT — embassy, RTIC para microcontroladores sem SO
  • Drivers de hardware — Google usa Rust no Android e ChromeOS
Web & Nuvem
  • WebAssembly — Rust compila para WASM, rodando no browser com desempenho nativo
  • Servidores HTTP — Actix-web e Axum lideram benchmarks de performance
  • CLIs & Ferramentas — ripgrep, fd, bat reescrevem utilitários Unix clássicos
Infraestrutura Crítica
  • Blockchain — Solana, Polkadot e Near Protocol são escritos em Rust
  • Bancos de dados — TiKV (banco chave-valor do TiDB) usa Rust
  • Segurança — Mozilla, Microsoft e Amazon migram partes críticas para eliminar CVEs de memória
